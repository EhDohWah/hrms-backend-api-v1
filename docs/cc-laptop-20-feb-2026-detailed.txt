  ---
  Key Business Rules

  1. Replacement logic ‚ÄĒ When new allocations replace old ones, the old
  allocation's end_date is set to new_start_date - 1 day (no overlap, no gap).
  2. Status + Date coupling ‚ÄĒ Status transitions in batchUpdate() automatically
  manage end_date:
    - Active ‚Üí Inactive: end_date = today
    - Inactive ‚Üí Active: end_date = null (cleared)
    - Deleted: status = Closed, end_date = today
  3. Import requires start_date ‚ÄĒ The import system requires start_date
  (validates as required|date), handles Excel serial number ‚Üí date conversion.
  4. History tracking ‚ÄĒ The observer tracks changes to both start_date and
  end_date in EmployeeFundingAllocationHistory, recording effective_date (from 
  allocation's start_date) and end_date for full audit trail.

  ---
  Frontend Note

  Currently, start_date and end_date are not displayed or editable in the 
  EmployeeFundingAllocationPanel.vue UI. The batch update flow sets start_date =
   today for new allocations and end_date = today for deletions automatically on
   the backend. The individual store/update endpoints (and import) accept dates
  from the request.

  The system uses both status and end_date to determine lifecycle:
  - status = the business meaning (active, paused, permanently closed)
  - end_date = the temporal boundary (when it stops being "current")
  - Together they determine: is this allocation effective right now?

  ---
  What the Frontend Does NOT Show

  Currently, the EmployeeFundingAllocationPanel.vue does not display or let 
  users edit start_date / end_date. The dates are managed automatically by the
  backend:
  - Batch create: start_date = today, end_date = null
  - Batch delete: end_date = today, status = Closed
  - Status toggle: end_date set/cleared automatically
  - Probation transitions: handled by daily scheduled job

  The store() and updateEmployeeAllocations() endpoints accept dates from the
  request, but those are not currently exposed in the UI panel.
